# منصة المقاول (El Moquwal) 🏗️ — دليل التشغيل والتطوير الشامل

المنصة الرقمية الأولى للمقاولات والتشطيبات في مصر. تجمع منصة **"المقاول"** بين أصحاب العقارات الباحثين عن خدمات تشطيب أو بناء، وبين نخبة من المهندسين والمقاولين المعتمدين في نظام آمن وشفاف يضمن جودة التنفيذ وحماية الحقوق المالية للجميع.

---

## 🛠️ المتطلبات الأساسية (Prerequisites)
تأكد من تثبيت وتوفير الأدوات التالية على جهازك قبل البدء:
1. **Node.js**: إصدار `18` أو أحدث (موصى باستخدام الإصدار المستقر `20` أو `22`).
2. **MongoDB**: قاعدة بيانات مستمرة. يمكنك إما:
   - تثبيتها محلياً وتفعيل خدمة الـ MongoDB Local Service.
   - أو استخدام قاعدة بيانات سحابية مجانية عبر **MongoDB Atlas**.
3. **Git**: لاستنساخ المستودع وتحديث الملفات.

## 🚀 أول مرة؟ كيفية تثبيت وتشغيل المشروع من الصفر (First-Time Setup)

اتبع الخطوات المرتبة التالية بعناية لإعداد المشروع على جهازك لأول مرة:

### 1. استنساخ المستودع والدخول للمجلد
افتح نافذة الترمينال (Terminal) واكتب:
```bash
# استنساخ المشروع من جيت هب
git clone https://github.com/0Marwan02/El-Moquwal.git

# الدخول إلى مجلد المشروع
cd el-moquwal
```

### 2. تثبيت المكتبات والاعتمادات (Dependencies)
تدار جميع المكتبات والاعتمادات البرمجية للمشروع (الباكيند والفرونتد) من خلال خادم الباكيند. قم بتثبيتها بتشغيل الأوامر التالية:
```bash
# الانتقال لمجلد الباكيند
cd backend

# تثبيت المكتبات (بما فيها Puppeteer لتوليد العقود، Argon2 لتشفير الباسوردات، وغيرها)
npm install
```

### 3. إعداد ملف متغيرات البيئة (Environment Variables)

2. انسخ محتويات الملف الاسترشادي `.env.example` وضعه داخل `.env` الجديد.
3. التكوين الافتراضي للملف يبدو كالتالي (ويمكنك تعديله إذا كانت بياناتك مختلفة):

### 4. ضخ البيانات الأولية للإدارة (Admin Seeding)
لكي تتمكن من الدخول للوحة التحكم والموافقة على المقاولين الجدد وحل النزاعات، يجب إنشاء حساب أدمن أساسي في قاعدة البيانات. 
بينما أنت داخل مجلد `backend/`، نفذ الأمر التالي:
```bash

```

## 💻 خيارات تشغيل المشروع محلياً (Running the App)

توفر المنصة طريقتين مرنتين للتشغيل حسب رغبتك كـ مطور أو مستخدم:

### 🟢 الطريقة الأولى: التشغيل المدمج على منفذ واحد (موصى بها للسهولة)
في هذه الطريقة، يقوم خادم الباكيند (Express) بتقديم ملفات الواجهة الأمامية بشكل ثابت ومباشر.
1. تأكد من تواجدك داخل مجلد `backend/`.
2. شغّل خادم المطور (الذي يعيد التشغيل تلقائياً عند تعديل أي ملف):
   ```bash
   npm run dev
   ```
3. افتح المتصفح وادخل إلى: 
   👉 **[http://localhost:4000](http://localhost:4000)**

### 🟡 الطريقة الثانية: التشغيل المنفصل عبر سكريبت البدء (مفيدة للتطوير المنفصل)
في هذه الطريقة، يعمل خادم الواجهة الأمامية على منفذ `3000` بشكل مستقل، ويعمل الباكيند على منفذ `4000` لحقن البيانات والـ APIs بشكل منفصل.
1. ارجع إلى المجلد الرئيسي للمشروع (المجلد الجذري `el-moquwal/`).
2. شغّل سكريبت البدء التلقائي:
   ```bash
   node start.js
   ```
3. افتح المتصفح وادخل للواجهة الأمامية عبر:
   👉 **[http://localhost:3000](http://localhost:3000)**

## ⚠️ حل المشاكل الشائعة (Troubleshooting)

*   **فشل الاتصال بقاعدة البيانات (Database Connection Error)**:
    تأكد من تشغيل خادم MongoDB محلياً على جهازك، أو تأكد من صحة رابط الـ `MONGO_URI` المكتوب بداخل ملف `.env` الخاص بك.
*   **خطأ في توليد العقد بصيغة PDF**:
    يستخدم النظام متصفح Puppeteer (Chromium) لتصوير وطباعة عقود الـ HTML العربية بدقة تامة. إذا واجهت مشكلة في التشغيل على سيرفرات بيئات معينة، سيقوم النظام تلقائياً بالتحول لـ Engine احتياطي مبني على مكتبة `pdfkit` لإتمام عمل العقد وتفادي تعطل الخدمة.
*   **ميزات الذكاء الاصطناعي لا تستجيب**:
    إذا لم تقم بتهيئة مفتاح API الخاص بأنثروبيك، فإن النظام مصمم برمجياً ليعمل في وضع المحاكاة (**Mock Mode**)، والذي يحسب الأسعار ويعطي مسودات وعروض تجريبية ذكية بشكل تلقائي ومجاني تماماً للتسهيل على المطورين والمستخدمين.
